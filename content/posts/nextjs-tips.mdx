## App Router의 기본 구조

Next.js 13부터 도입된 App Router는 `app/` 디렉토리를 기반으로 동작합니다.

### 서버 컴포넌트와 클라이언트 컴포넌트

기본적으로 모든 컴포넌트는 **서버 컴포넌트**입니다. 브라우저 API나 React hooks가 필요하면 `"use client"` 지시어를 추가합니다.

```tsx
// 서버 컴포넌트 (기본값)
async function ServerComponent() {
  const data = await fetch("https://api.example.com/data");
  return <div>{/* ... */}</div>;
}

// 클라이언트 컴포넌트
"use client";
import { useState } from "react";

function ClientComponent() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

### 동적 라우트

`[slug]` 형식의 폴더로 동적 경로를 처리합니다.

```
app/
  blog/
    [slug]/
      page.tsx  → /blog/my-post
```

## 데이터 패칭

App Router에서는 `async/await`를 컴포넌트 함수 내부에서 바로 사용할 수 있습니다.

```tsx
async function PostPage({ params }: { params: { slug: string } }) {
  const post = await getPost(params.slug);
  return <article>{post.content}</article>;
}
```

정적 사이트 생성이 필요하다면 `generateStaticParams`를 함께 사용합니다.
